## 摘要

本课程设计基于JSP+Servlet+JavaBean+JDBC+MySQL技术栈，采用MVC分层架构设计并实现了一个功能完整的电子商务平台。系统包含后台管理子系统和前台电子商务子系统两大模块。后台面向管理员提供商品管理、分类管理、订单管理、用户管理和销量统计等功能；前台面向普通用户提供商品浏览、搜索、购物车管理、订单生成、模拟支付、商品收藏和个人信息维护等功能，未登录游客仅可浏览和搜索商品。项目使用Maven构建，基于Jakarta Servlet API 6.1适配Tomcat 11服务器，JSTL 3.0标签库实现视图层数据渲染，前端采用Bootstrap 5框架配合自定义CSS实现响应式布局。系统严格按照MVC设计模式，实现了表示层（JSP）、控制层（Servlet）、业务逻辑层（JavaBean）和数据访问层（DAO）的清晰分离，完成了7张数据库表的设计与实现，具备完善的权限控制、SHA-256加盐密码加密和Session会话管理机制。

**关键词：** Java Web；电子商务平台；JSP；Servlet；JDBC；MVC；MySQL

## 第1章 绪论

### 1.1 设计背景

随着互联网技术的飞速发展和电子商务的普及，在线购物已经成为人们日常生活中不可或缺的一部分。从大型综合电商平台到垂直领域的专业电商，电子商务系统承载着海量商品信息的展示、用户交互、订单处理和支付结算等核心业务功能。作为计算机相关专业的学生，掌握Web应用系统的开发方法，特别是Java Web技术栈的实际应用，是课程学习的重要目标。

Java Web程序设计课程涵盖了JSP、Servlet、JavaBean、JDBC等核心技术，是Java EE开发体系的基础。本次课程设计以"电子商务平台"为选题，旨在通过一个完整的项目实践，将课堂所学的理论知识转化为实际开发能力。该选题具有明确的业务场景、清晰的用户角色划分和完善的业务流程，非常适合作为课程设计的综合训练项目。

### 1.2 设计目的

通过本次课程设计，达到以下目标：

1. 掌握基于JSP、JavaBean、Servlet的Web应用开发流程、方法和基本技术。
2. 理解MVC分层思想，能够完成表示层、控制层、业务逻辑层和数据访问层的基本设计。
3. 掌握Servlet请求处理、JSP页面展示、JavaBean数据封装、JDBC数据访问等核心技术。
4. 熟悉Web应用系统的需求分析、功能设计、数据库设计和编码实现过程。
5. 能够完成用户登录、权限控制、数据增删改查、条件查询、分页显示、表单验证等常见功能。
6. 培养系统设计能力、编码开发能力、问题调试能力和文档撰写能力。

### 1.3 开发环境

本系统的开发环境配置如下：

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows 10/11 |
| JDK版本 | OpenJDK 17 |
| Web服务器 | Apache Tomcat 11（支持Jakarta Servlet 6.1） |
| 数据库 | MySQL 8.0 |
| 开发工具 | IntelliJ IDEA |
| 构建工具 | Maven 3.8+ |
| 前端框架 | Bootstrap 5.3 + 自定义CSS/JS |

特别说明：本项目使用**Jakarta Servlet API 6.1**，适配Tomcat 11服务器。Tomcat 10及以上版本将`javax.*`命名空间迁移至`jakarta.*`，因此项目中所有Servlet、JSP、JSTL、Filter等组件均使用Jakarta命名空间。密码存储采用**SHA-256加随机盐**的哈希方案，数据库密码字段存储格式为`base64(salt):hex(hash)`。

### 1.4 系统主要功能简介

本电子商务平台系统分为两大子系统：

**（一）后台管理子系统**

管理员登录成功后进入后台管理页面，能够对商品、商品类型、注册用户和订单信息进行管理。主要功能包括：
- 管理员登录与安全退出
- 商品管理：新增商品、删除商品、修改商品、查询商品、商品库存维护
- 商品类型管理：新增类型、删除类型、修改类型、查询类型
- 订单管理：订单查询、订单详情查看、订单状态修改
- 用户管理：注册用户查询、用户状态管理
- 销量统计：按月份统计销售数据、按商品类型统计销售数据

**（二）电子商务前台子系统**

面向普通用户和未登录用户提供不同功能：
- 未登录（游客）：浏览商品、搜索商品、查看商品详情
- 已登录用户：商品浏览与搜索、加入购物车、购买商品、模拟支付、查看订单、查看收藏、维护个人信息、安全退出

## 第2章 需求分析

### 2.1 用户角色分析

本系统涉及两类用户角色，具有不同的系统访问权限和操作范围：

**1. 管理员（Admin）**

管理员是系统的后台管理者，负责维护整个电商平台的正常运行。管理员的职责包括：
- 管理商品信息（上架、下架、定价、库存维护）
- 管理商品分类体系
- 监管用户订单，处理订单状态变更
- 管理注册用户账户，控制用户状态
- 查看销售统计数据，辅助经营决策

管理员通过专用登录入口进入后台管理系统，系统通过role字段（值为1）标识管理员身份。

**2. 普通用户/会员（User）**

普通用户是系统的主要使用者，通过注册成为会员后可使用全部前台功能：
- 浏览和搜索商品
- 将商品加入购物车
- 提交订单并模拟支付
- 收藏感兴趣的商品
- 管理个人信息和密码

用户通过role字段（值为0）标识，status字段（值为1表示正常，0表示禁用）控制账户状态。

**3. 未登录游客（Guest）**

游客无需注册即可访问系统前台，但功能受限：
- 浏览商品列表
- 按分类筛选商品
- 搜索商品
- 查看商品详情

游客不能进行购物车操作、下单、收藏和个人信息管理。

### 2.2 功能需求分析

**（一）用户注册与登录模块**

- 用户注册：新用户填写用户名、密码、确认密码、真实姓名、邮箱、电话、地址等信息进行注册
- 用户名唯一性校验：系统检查用户名是否已被注册
- 用户登录：输入用户名和密码进行身份验证
- 角色区分：登录后根据role字段自动跳转至前台首页或后台管理页面
- 安全退出：清除Session实现安全退出

**（二）商品浏览与搜索模块**

- 商品分类浏览：按商品分类筛选显示商品列表
- 分页展示：商品列表支持分页，每页显示12个商品
- 关键词搜索：支持按商品名称和描述进行模糊搜索
- 商品详情：展示商品的完整信息，包括图片、名称、价格、库存、描述等
- 首页推荐：首页展示热门商品和各分类下的推荐商品

**（三）购物车管理模块**

- 添加商品：从商品详情页将商品加入购物车
- 修改数量：在购物车中修改商品购买数量
- 删除商品：从购物车中移除商品
- 购物车汇总：显示购物车商品总计金额
- 结算：填写收货信息后提交订单

**（四）订单管理模块**

- 订单生成：从购物车商品生成订单，同时扣减库存，清空购物车
- 订单列表：分页查看个人历史订单
- 订单详情：查看订单基本信息、收货信息和商品明细
- 模拟支付：对"待付款"状态的订单进行模拟支付操作
- 订单状态流转：待付款→已付款→已发货→已完成

**（五）商品收藏模块**

- 添加收藏：已登录用户可将商品加入收藏夹
- 查看收藏：在个人收藏页面查看所有收藏商品
- 取消收藏：从收藏列表中移除商品

**（六）个人信息管理模块**

- 查看个人信息：展示当前登录用户的基本信息
- 修改个人信息：更新真实姓名、邮箱、电话、地址
- 修改密码：验证旧密码后设置新密码

**（七）后台商品管理模块**

- 商品列表：分页展示所有商品，支持分类筛选和关键词搜索
- 新增商品：填写商品名称、分类、价格、库存、图片、描述等信息
- 编辑商品：修改已有商品的全部信息
- 删除商品：从系统中删除商品
- 上架/下架管理：控制商品的展示状态

**（八）后台分类管理模块**

- 分类列表：展示所有商品分类
- 新增分类：添加新的商品分类
- 编辑分类：修改分类名称和描述
- 删除分类：删除分类（级联删除关联商品）
- 分类商品数量统计：显示每个分类下的商品数量

**（九）后台订单管理模块**

- 订单列表：分页展示所有订单，支持状态筛选和关键词搜索
- 订单详情：查看订单完整信息
- 订单状态变更：管理员修改订单状态

**（十）后台用户管理模块**

- 用户列表：分页展示所有注册用户
- 用户状态管理：启用或禁用用户账户

**（十一）销量统计模块**

- 按月统计：统计各月份的销售额（已付款/已发货/已完成状态的订单）
- 按分类统计：统计各类别商品的销售额

### 2.3 非功能需求分析

1. **性能需求**：系统应在3秒内响应用户请求，支持至少50个并发用户的正常访问。
2. **安全性需求**：使用Session管理用户登录状态，Session超时设置为30分钟；管理员操作需验证权限；密码采用SHA-256加随机盐哈希存储，防止明文泄露风险。
3. **可用性需求**：界面友好美观，采用Bootstrap 5响应式布局，适配不同屏幕尺寸。
4. **编码规范**：统一使用UTF-8字符编码，避免中文乱码问题。
5. **可维护性**：采用MVC分层架构，代码结构清晰，便于后续维护和扩展。
6. **数据完整性**：通过外键约束保证数据一致性，使用事务处理订单创建等关键操作。

本系统采用JSP+Servlet+JavaBean+JDBC+MySQL技术栈，这是Java Web开发领域最为成熟和经典的技术组合。所有技术组件均为开源免费产品，社区生态完善，文档资源丰富。开发工具IntelliJ IDEA为业界主流IDE，Maven作为依赖管理和构建工具，Bootstrap 5作为前端框架，这些技术的组合完全能够满足电子商务平台的开发需求。

## 第3章 系统设计

### 3.1 系统总体架构

本系统采用经典的MVC（Model-View-Controller）分层架构，各层职责明确，耦合度低：

```
┌─────────────────────────────────────────────────┐
│              表示层 (View)                        │
│         JSP页面 + Bootstrap 5 + EL/JSTL          │
│         负责数据展示和用户交互                      │
├─────────────────────────────────────────────────┤
│              控制层 (Controller)                  │
│    Servlet（@WebServlet注解配置）                  │
│   负责请求接收、参数处理、流程控制和页面跳转            │
├─────────────────────────────────────────────────┤
│              业务逻辑层 (Model)                     │
│        JavaBean（实体类）+ DAO（数据访问对象）        │
│       负责数据封装和业务逻辑处理                     │
├─────────────────────────────────────────────────┤
│              数据访问层 (DAO)                      │
│          JDBC + PreparedStatement                │
│         负责数据库的CRUD操作                       │
├─────────────────────────────────────────────────┤
│              数据层 (Database)                    │
│              MySQL 8.0                           │
│              7张业务数据表                         │
└─────────────────────────────────────────────────┘
```

**各层职责说明**：

- **表示层**：JSP页面负责HTML渲染和数据展示，使用JSTL标签库和EL表达式动态展示数据，避免在JSP中编写大量Java代码。Bootstrap 5提供响应式布局和美观的UI组件。
- **控制层**：通过@WebServlet注解配置的Servlet接收HTTP请求，处理参数、调用DAO层执行业务逻辑，将结果放入request/session作用域，通过转发或重定向控制页面跳转。
- **业务逻辑层**：JavaBean实体类封装数据对象（User、Product、Order等），DAO层封装数据访问逻辑，实现业务数据的CRUD操作。
- **数据访问层**：使用JDBC和PreparedStatement执行SQL语句，DBUtil工具类统一管理数据库连接，防止SQL注入攻击。
- **数据层**：MySQL数据库存储系统的所有业务数据。

### 4.2 数据库连接实现

数据库连接由`DBUtil`工具类统一管理，使用静态代码块在类加载时读取`db.properties`配置文件并注册数据库驱动。

**db.properties配置文件**：

```properties
jdbc.driver=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/db_ecommerce?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf-8
jdbc.username=root
jdbc.password=123456
```

**DBUtil核心实现**：

```java
public class DBUtil {
    private static String driver;
    private static String url;
    private static String username;
    private static String password;

    static {
        try {
            Properties props = new Properties();
            InputStream is = DBUtil.class.getClassLoader()
                .getResourceAsStream("db.properties");
            props.load(is);
            driver = props.getProperty("jdbc.driver");
            url = props.getProperty("jdbc.url");
            username = props.getProperty("jdbc.username");
            password = props.getProperty("jdbc.password");
            Class.forName(driver);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(url, username, password);
    }

    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        try { if (rs != null) rs.close(); } catch (SQLException e) {}
        try { if (stmt != null) stmt.close(); } catch (SQLException e) {}
        try { if (conn != null) conn.close(); } catch (SQLException e) {}
    }
}
```

设计要点：
- 静态代码块确保数据库驱动只注册一次
- 连接URL中配置了`useSSL=false`、`serverTimezone=Asia/Shanghai`和`characterEncoding=utf-8`，确保时间和中文编码正确
- 提供重载的close方法，灵活关闭不同组合的资源
- 每个DAO方法在finally块中调用DBUtil.close()确保连接释放

### 4.3 登录与权限控制实现

**（一）用户登录（LoginServlet）**

登录处理流程如下：

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    private final UserDao userDao = new UserDao();

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {
        String username = req.getParameter("username");
        String password = req.getParameter("password");

        // 1. 查询用户
        User user = userDao.findByUsername(username);
        if (user == null) {
            req.setAttribute("msg", "用户名不存在");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }
        // 2. 验证密码（使用SHA-256加盐哈希验证）
        if (!PasswordUtil.verify(password, user.getPassword())) {
            req.setAttribute("msg", "密码错误");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }
        // 3. 检查状态
        if (user.getStatus() != 1) {
            req.setAttribute("msg", "账号已被禁用");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }
        // 4. 设置Session
        HttpSession session = req.getSession();
        session.setAttribute("user", user);
        // 5. 按角色跳转
        if (user.getRole() == 1) {
            resp.sendRedirect(req.getContextPath() + "/admin/index.jsp");
        } else {
            resp.sendRedirect(req.getContextPath() + "/index.jsp");
        }
    }
}
```

**（二）权限控制机制**

系统采用三层权限控制策略：

1. **Filter层**：EncodingFilter统一设置字符编码为UTF-8。

2. **Servlet层**：所有后台Servlet在方法开头进行管理员权限校验：
```java
User user = (User) session.getAttribute("user");
if (user == null || user.getRole() != 1) {
```

3. **JSP层**：前台需要登录的页面使用JSTL标签判断：

后台页面双重校验：
```jsp
<c:if test="${sessionScope.user == null || sessionScope.user.role != 1}">

</c:if>
```

**（二）订单创建（事务处理）**

订单创建是系统的核心业务流程，涉及多表操作，必须使用事务保证数据一致性：

```java
public void createOrder(Order order) throws SQLException {
    Connection conn = null;
    try {
        conn = DBUtil.getConnection();
        conn.setAutoCommit(false);  // 开启事务

        // 1. 生成订单编号（UUID前32位）
        String orderNo = UUID.randomUUID().toString()
            .replace("-", "").substring(0, 32);

        // 2. 插入订单主表
        String orderSql = "INSERT INTO t_order(order_no, user_id, ...) VALUES(?,?,...)";
        PreparedStatement ps = conn.prepareStatement(orderSql, 
            Statement.RETURN_GENERATED_KEYS);
        ps.setString(1, orderNo);
        // ... 设置其他参数
        ps.executeUpdate();

        // 获取自增ID
        ResultSet rs = ps.getGeneratedKeys();
        int orderId = rs.next() ? rs.getInt(1) : 0;

        // 3. 批量插入订单明细
        String itemSql = "INSERT INTO t_order_item(...) VALUES(?,?,?,?,?,?)";
        ps = conn.prepareStatement(itemSql);
        for (OrderItem item : order.getItems()) {
            ps.setInt(1, orderId);
            ps.setInt(2, item.getProductId());
            // ... 设置其他参数
            ps.addBatch();
        }
        ps.executeBatch();

        conn.commit();  // 提交事务
    } catch (SQLException e) {
        if (conn != null) conn.rollback();  // 回滚事务
        throw e;
    } finally {
        if (conn != null) {
            conn.setAutoCommit(true);
            DBUtil.close(conn, null);
        }
    }
}
```

**（三）购物车管理**

购物车的数据操作在`CartServlet`中处理，通过不同的action参数区分操作类型：

- `action=add`：用户点击"加入购物车"，先检查是否已存在该商品，存在则增加数量，否则新增
- `action=update`：用户修改数量，提交表单更新
- `action=delete`：用户移除购物车商品

购物车列表通过JOIN查询关联商品信息，直接获取商品名称、价格和图片。

**（四）模拟支付**

支付流程在`OrderServlet`中通过`action=pay`处理：

1. 验证当前用户是否为订单所属人
2. 检查订单状态是否为"待付款"（status=0）
3. 调用`OrderDao.updateStatusWithPayTime(id, 1)`更新状态为"已付款"并记录支付时间

**（五）销量统计**

`OrderDao`提供两个统计方法：

- `getSalesByMonth()`：使用`DATE_FORMAT(create_time, '%Y-%m')`按月分组，`SUM(total_amount)`汇总已付款/已发货/已完成状态的订单金额
- `getSalesByCategory()`：通过`LEFT JOIN`关联分类表、商品表、订单明细表和订单表，按分类分组统计销售额

### 4.5 关键代码说明

**（一）分页查询实现**

所有列表页面统一采用以下分页参数：
- `currentPage`：当前页码（默认1）
- `pageSize`：每页显示条数（默认12，后台默认10）
- `totalCount`：总记录数
- `totalPages`：总页数（=(totalCount + pageSize - 1) / pageSize）

SQL分页使用`LIMIT offset, pageSize`，其中`offset = (page - 1) * pageSize`。

**（二）字符编码过滤器**

`EncodingFilter`实现`Filter`接口，对所有请求进行UTF-8编码设置：

```java
@WebFilter("/*")
public class EncodingFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {
        request.setCharacterEncoding("UTF-8");
        response.setContentType("text/html;charset=UTF-8");
        response.setCharacterEncoding("UTF-8");
        chain.doFilter(request, response);
    }
}
```

**（三）密码加密方案**

用户密码采用SHA-256加随机盐的哈希方案，通过`PasswordUtil`工具类实现：

- **加密存储**：注册时调用`PasswordUtil.hash(plainPassword)`生成格式为`base64(salt):hex(hash)`的密文存入数据库
- **登录验证**：调用`PasswordUtil.verify(plainPassword, storedHash)`提取盐值后对输入密码进行相同哈希运算，比对结果
- **安全性**：每个用户拥有独立的16字节随机盐值，即使两个用户设置相同密码，数据库中存储的密文也完全不同，有效防御彩虹表攻击

**（四）Bootstrap响应式布局与自定义样式**

所有JSP页面通过CDN引入Bootstrap 5.3，配合项目自定义的`css/style.css`（174行）和`js/common.js`（59行）实现完整的用户界面：

- 导航栏使用`navbar-expand-lg`实现响应式折叠
- 商品列表使用`col-md-3 col-sm-6`实现4列到2列自适应
- 表格使用`table-responsive`实现横向滚动
- 后台管理使用`flex`布局实现固定250px侧边栏
- 自定义CSS定义了商品卡片悬浮动效、轮播图高度、状态标签配色（待付款橙色/已付款蓝色/已发货绿色/已完成深绿/已取消灰色）、统计卡片样式等
- 自定义JS实现了AJAX加入购物车、收藏切换、购物车数量变更等前端交互功能

### 6.2 存在问题

1. **未实现验证码功能**：登录和注册页面缺少图形验证码，存在被恶意暴力破解或批量注册的风险。

2. **未实现订单超时自动取消**：对于长时间未付款的订单，系统没有自动取消的机制，可能导致库存长期锁定。

3. **支付为模拟支付**：支付功能仅修改订单状态，未接入真实支付接口（如支付宝、微信支付）。

4. **未使用连接池**：当前每次数据库操作都新建连接（通过DriverManager直接获取），高并发场景下性能较差，应使用Druid或HikariCP连接池技术。

5. **缺少日志系统**：系统未引入日志框架（如SLF4J/Log4j2），异常信息仅通过`e.printStackTrace()`输出，不利于生产环境的问题排查和监控。

6. **前后端未完全分离**：部分JSP页面（如首页、商品列表页）中仍包含少量Java脚本（scriptlet），未完全遵循纯MVC模式，增加了维护复杂度。

### 6.3 改进方向

1. 添加图形验证码功能，提升登录和注册接口的安全性。
2. 使用定时任务框架（如Quartz或Spring Task）实现订单超时自动取消。
3. 引入数据库连接池技术（如Druid或HikariCP），提升系统在高并发场景下的性能表现。
4. 引入日志框架（SLF4J + Log4j2），建立完善的日志记录体系。
5. 使用Redis缓存热门商品和首页数据，减少数据库查询压力。
6. 将JSP页面中的scriptlet代码迁移至Servlet或自定义标签，实现纯MVC架构。
7. 引入Spring框架（Spring MVC + Spring JDBC / MyBatis）简化开发流程，降低代码耦合度。
8. 实现商品多图展示、商品规格（SKU）等高级电商功能。

**（一）DBUtil.java - 数据库连接工具类**

```java
package com.ecommerce.util;

import java.io.IOException;
import java.io.InputStream;
import java.sql.*;
import java.util.Properties;

public class DBUtil {
    private static String driver;
    private static String url;
    private static String username;
    private static String password;

    static {
        Properties props = new Properties();
        try (InputStream is = DBUtil.class.getClassLoader()
                .getResourceAsStream("db.properties")) {
            if (is != null) {
                props.load(is);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        driver = props.getProperty("jdbc.driver", "com.mysql.cj.jdbc.Driver");
        url = props.getProperty("jdbc.url");
        username = props.getProperty("jdbc.username");
        password = props.getProperty("jdbc.password");
        try {
            Class.forName(driver);
        } catch (ClassNotFoundException e) {
            e.printStackTrace();
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(url, username, password);
    }

    /** 关闭数据库资源 */
    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        if (rs != null) { try { rs.close(); } catch (SQLException ignored) {} }
        if (stmt != null) { try { stmt.close(); } catch (SQLException ignored) {} }
        if (conn != null) { try { conn.close(); } catch (SQLException ignored) {} }
    }

    public static void close(Connection conn, Statement stmt) {
        close(conn, stmt, null);
    }
}
```

**（二）LoginServlet.java - 登录处理（含密码哈希验证）**

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    private final UserDao userDao = new UserDao();

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {
        req.getRequestDispatcher("/login.jsp").forward(req, resp);
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {
        req.setCharacterEncoding("UTF-8");

        String username = req.getParameter("username");
        String password = req.getParameter("password");

        User user = userDao.findByUsername(username);
        if (user == null) {
            req.setAttribute("msg", "用户名不存在");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }
        if (!PasswordUtil.verify(password, user.getPassword())) {
            req.setAttribute("msg", "密码错误");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }
        if (user.getStatus() != 1) {
            req.setAttribute("msg", "账号已被禁用");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }

        HttpSession session = req.getSession();
        session.setAttribute("user", user);

        if (user.getRole() == 1) {
            resp.sendRedirect(req.getContextPath() + "/admin/index.jsp");
        } else {
            resp.sendRedirect(req.getContextPath() + "/index.jsp");
        }
    }
}
```

**（三）OrderDao.createOrder() - 订单创建事务处理**

```java
public void createOrder(Order order) throws SQLException {
    Connection conn = null;
    try {
        conn = DBUtil.getConnection();
        conn.setAutoCommit(false);

        String orderNo = UUID.randomUUID().toString()
            .replace("-", "").substring(0, 32);
        String orderSql = "INSERT INTO t_order(order_no, user_id, total_amount, "
            + "status, receiver_name, receiver_phone, receiver_address, create_time) "
            + "VALUES(?,?,?,?,?,?,?,NOW())";
        PreparedStatement ps = conn.prepareStatement(orderSql, 
            Statement.RETURN_GENERATED_KEYS);
        ps.setString(1, orderNo);
        ps.setInt(2, order.getUserId());
        ps.setDouble(3, order.getTotalAmount());
        ps.setInt(4, order.getStatus());
        ps.setString(5, order.getReceiverName());
        ps.setString(6, order.getReceiverPhone());
        ps.setString(7, order.getReceiverAddress());
        ps.executeUpdate();

        ResultSet rs = ps.getGeneratedKeys();
        int orderId = rs.next() ? rs.getInt(1) : 0;
        DBUtil.close(null, ps, rs);

        if (order.getItems() != null) {
            String itemSql = "INSERT INTO t_order_item(order_id, product_id, "
                + "product_name, product_price, quantity, subtotal) "
                + "VALUES(?,?,?,?,?,?)";
            ps = conn.prepareStatement(itemSql);
            for (OrderItem item : order.getItems()) {
                ps.setInt(1, orderId);
                ps.setInt(2, item.getProductId());
                ps.setString(3, item.getProductName());
                ps.setDouble(4, item.getProductPrice());
                ps.setInt(5, item.getQuantity());
                ps.setDouble(6, item.getSubtotal());
                ps.addBatch();
            }
            ps.executeBatch();
            DBUtil.close(null, ps);
        }

        conn.commit();
        order.setId(orderId);
        order.setOrderNo(orderNo);
    } catch (SQLException e) {
        if (conn != null) conn.rollback();
        throw e;
    } finally {
        if (conn != null) {
            conn.setAutoCommit(true);
            DBUtil.close(conn, null);
        }
    }
}
```
